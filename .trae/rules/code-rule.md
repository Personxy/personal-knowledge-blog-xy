# Code Generation Rules

## 1. 交互规范
- **语言**: 强制中文回答。
- **代码注释**: 关键逻辑必加中文注释；代码块 >20 行优先拆分/抽象。

## 2. 编码原则 (General)
- **简洁性**: 避免深层嵌套 (Early Return)；避免无效对象复制。
- **命名**: 描述性命名，禁止无意义缩写 (除 `i`, `err` 等惯例)。
- **职责**: 单一职责原则 (函数只做一件事)。

## 3. 重构与质量
- **流程**: 小步修改 -> 验证 -> 提交。
- **安全**: 修改前确认现有逻辑；修改后验证行为一致性。

## 4. 性能考量
- **内存**: 及时释放资源，避免内存泄漏。
- **计算**: 避免循环内重复计算；合理使用数据结构。
- **并发**: 识别并行机会；注意线程安全 (Backend)。
